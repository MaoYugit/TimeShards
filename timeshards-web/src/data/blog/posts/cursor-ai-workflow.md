这篇整理我自己用得比较顺手的一套流程，适合个人项目和 2~5 人小团队。

## 目标

让 AI 参与开发，但不牺牲可维护性：

- 需求明确
- 变更可追踪
- 验收可执行

## 我的工作流

### 1) 先定义“可验收结果”

在动手前，我会先写三条：

1. 页面/功能最终行为是什么。
2. 边界条件有哪些。
3. 如何快速验证（手测 + 构建）。

### 2) 给 AI 足够上下文

- 当前文件
- 关联数据结构
- 路由/状态管理位置

上下文不完整时，AI 代码很容易“看起来对，实际上跑不通”。

### 3) 最后做一轮收口

- `pnpm run build` 或 `npm run build`
- 快速点一遍主路径
- 检查移动端布局

## 结论

AI 最适合加速“明确任务”，而不是替你做需求决策。  
把决策留给人，把执行交给工具，效率会更稳定。
