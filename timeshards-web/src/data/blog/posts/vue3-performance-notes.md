这篇是我在项目里最常用的一套优化顺序，目标不是“极限跑分”，而是先把用户能感知的卡顿解决掉。

## 1. 先找最贵的更新路径

建议先用 DevTools 看三件事：

- 哪个组件重复渲染最多。
- 哪次交互导致了大面积更新。
- 列表节点是否频繁销毁重建。

如果你还不知道慢在哪里，就不要急着做“高级优化”。

## 2. 列表场景优先做这几条

```ts
// 关键点：稳定 key + 拆分子组件 + 控制响应式范围
const items = shallowRef<Row[]>([])

function updateRows(next: Row[]) {
  items.value = next
}
```

- 尽量保证 `key` 稳定，避免整段重建。
- 大对象列表优先 `shallowRef`，减少深层代理成本。
- 把“频繁变化的状态”下沉到更小组件里。

## 3. 动画与滚动

- 仅在必要元素上使用 `will-change`。
- 避免在滚动中触发布局抖动（比如频繁读写同一批布局属性）。
- 能用 `transform/opacity` 的尽量不用 `top/left`。

## 4. 一条经验

优化最有效的方式通常不是“加技术”，而是**删掉不必要的更新**。
