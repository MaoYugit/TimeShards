## 全局背景（`timeshards-web`）

这个全局背景由两层叠加组成：

- **Three.js**：3D「时间碎片」(Tetrahedron) 漂浮 + 自转
- **tsParticles**：2D 连线粒子（点击生成、悬停排斥）

背景组件在 `App.vue` 顶层挂载，固定铺满全屏，且 **`pointer-events: none`**，不会影响页面交互。

---

### 目录与文件

- **组件**：`timeshards-web/src/components/Background/GlobalBackground.vue`
- **插件注册**：`timeshards-web/src/main.ts`
- **挂载位置**：`timeshards-web/src/App.vue`

---

### 依赖

`timeshards-web/package.json` 需要包含：

- `three`
- `@tsparticles/vue3`
- `@tsparticles/slim`
- `@types/three`（devDependency）

### tsParticles 注册方式（全局一次）

`@tsparticles/vue3` v3 会全局注册组件标签名 **`vue-particles`**。

当前项目采用在入口统一初始化引擎（推荐做法：只初始化一次，组件只传 options）：

- 文件：`timeshards-web/src/main.ts`
- 做法：在 `VueParticles(app, { init })` 中调用 `loadSlim(engine)`

这样 `GlobalBackground.vue` 内只需要：

- `<vue-particles :options="particleOptions" />`
- 可选监听：`@particlesLoaded`

---

### 连线密度与“渐进式清理”

为了让桌面端连线更丰富、同时避免点击后粒子无限增长，当前策略是：

- **桌面端（fine pointer）**：基础粒子数量 `240`
- **移动端（coarse pointer）**：基础粒子数量 `110`（更省电、更稳帧）
- **点击 push 产生的额外粒子**：会被定时“渐进式清理”
  - 每 `450ms` 少量移除（每次最多 `3` 个）
  - 只清理“超过基础数量”的部分，基础粒子不会被清掉

### Three.js 结构与动画逻辑

`GlobalBackground.vue` 内部做了这些事：

- **场景与相机**
  - `scene = new THREE.Scene()`
  - `scene.fog = new THREE.FogExp2(...)` 增强纵深
  - `camera = new THREE.PerspectiveCamera(...)`，`camera.position.z = 50`
- **渲染器**
  - `new THREE.WebGLRenderer({ alpha: true, antialias: true })`
  - `alpha: true` 保证 canvas 背景透明，让 CSS 渐变底色可见
  - `setPixelRatio(Math.min(devicePixelRatio, 2))` 控制高 DPR 屏性能
- **碎片（50 个）**
  - `THREE.TetrahedronGeometry(1, 0)`
  - `THREE.MeshPhongMaterial({ flatShading: true, transparent: true, opacity: 0.6 })`
  - 每个 Mesh 随机位置 / 随机旋转 / 随机缩放
  - 通过 `userData` 保存每个碎片自己的 `rotationSpeed` 与 `floatSpeed`
- **动画循环**
  - `requestAnimationFrame` 每帧更新：
    - 自转：`rotation.x/y += rotationSpeed`
    - 漂浮：`position.y += floatSpeed`
    - 边界回卷：超过 `±50` 从另一端出现（无限漂浮效果）
- **resize**
  - 更新 `camera.aspect` + `renderer.setSize`

### 层级与样式约定

组件固定全屏铺底：

- `.background-wrapper`：`position: fixed; inset: 0; z-index: -1; overflow: hidden`
- `.three-container`：z-index 0
- `.particles-container`：z-index 1（粒子层在 3D 上面）
- `.background-wrapper`：`pointer-events: none`（不挡点击/滚动/hover）

---

### 释放资源（避免内存泄漏）

组件卸载时做了清理（`onBeforeUnmount`）：

- `cancelAnimationFrame`
- 移除 `resize` 监听
- `geometry.dispose()` / `material.dispose()`
- `renderer.dispose()` 并把 canvas 从 DOM 移除

如果未来把全局背景改成可切换/可销毁，这一步是必须的，否则 WebGL 资源会累积导致卡顿甚至崩溃。

---

### 常见问题排查

- **看不到粒子**
  - 确认 `main.ts` 已执行 `VueParticles(app, { init })`
  - 确认模板使用的是 `<vue-particles ... />`（不是 `<Particles />`）
- **背景遮住页面无法点击**
  - 确认 `.background-wrapper { pointer-events: none; }`
  - 确认 `z-index: -1`（背景在内容下面）
- **3D 画面全黑/渐变被盖住**
  - 确认 `WebGLRenderer({ alpha: true })`
  - 确认粒子 options `background` 是 `transparent`
